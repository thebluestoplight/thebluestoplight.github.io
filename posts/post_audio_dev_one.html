Introduction

Today, I continued a project I already started a couple months back, which is a synthesizer plugin made in the JUCE Framework. I also decided to re-learn how to use Max for Live (M4L), so I followed along a video, and made some minor changes.

JUCE Synth

I'm learning JUCE as well, but I think I have a better understanding of it (at least for now) than M4L. Because of this, as a first step I'm following along a tutorial series by TheAudioProgrammer. I'm planning on extending the plugin after I finished laying down the basics.

Today I completed tutorial #4 and #5. I didn't make any notable changes, given that these videos are mostly about the back-end of the plugin; fixing the oscillator making a pop sound on MIDI input, and connecting the AudioParameterValueStreeState (APVTS) with the parameters and the UI. Although some code is related to adding sliders to the GUI, there wasn't really any styling involved.

Here are some code snippets from today's session:

                
    void SynthVoice::updateADSR(const float attack, const float decay, const float sustain, const float release)
    {
        adsrParams.attack = attack;
        adsrParams.decay = decay;
        adsrParams.sustain = sustain;
        adsrParams.release = release;
    
        adsr.setParameters(adsrParams);
    }
                
            
                
    void AkiAudioProcessorEditor::resized()
    {
        const auto bounds = getLocalBounds().reduced(10);
        const auto padding = 10;
    
        const auto sliderWidth = bounds.getWidth() / 4 - padding;
        const auto sliderHeight = bounds.getWidth() / 4 - padding;
        const auto sliderStartX = 0;
        const auto sliderStartY = bounds.getHeight() / 2 - (sliderHeight / 2);
    
        attackSlider.setBounds(sliderStartX, sliderStartY, sliderWidth, sliderHeight);
        decaySlider.setBounds(attackSlider.getRight() + padding, sliderStartY, sliderWidth, sliderHeight);
        sustainSlider.setBounds(decaySlider.getRight() + padding, sliderStartY, sliderWidth, sliderHeight);
        releaseSlider.setBounds(sustainSlider.getRight() + padding, sliderStartY, sliderWidth, sliderHeight);
    }

This code is identical to the one shown in the video. In case you're interested in making a plugin yourself, I highly recommend checking out TheAudioProgrammer channel, and the AudioProgrammer community.

Max for Live FM Synth

Yesterday I had the idea to make some M4L devices too. I thought it would be a pretty neat thing to make effects and instruments for Ableton, since I use it all the time anyway.

Back in 2020 my friend instructed me on how to make my first M4L device, which was a ringmod effect. Since then I forgot everything about Max.

I followed along this video after rediscovering the basics. I made next to no changes with the code itself because I don't know how, but decided to keep the two scopes in my final product that were deleted in the video.

My synth is also, not polyphonic. For some reason I had no audio output at the end, after implementing polyphony, so I reverted the changes to have a working instrument because I'm not sure I' d like to work anymore on this particular project.

Conclusion

This was productive day, although this isn't my achievement - instead I stayed in tutorial hell for a bit more time. Nevertheless, I'm glad I started making Max for Live devices, and I'm definitely planning on making more in the future, alongside with more VST plugin with JUCE.
